
# 자동차 경주 게임

여러 자동차가 경주하는 콘솔 게임입니다.  
사용자가 자동차 이름과 시도 횟수를 입력하면 랜덤으로 자동차가 전진하고, 최종 우승자를 알려줍니다.

---

## 학습 목표
- 여러 역할을 수행하는 큰 함수를 단일 책임 메서드로 분리
- JUnit 5, AssertJ를 이용한 테스트 작성 및 검증
- 코드 가독성, Indent depth 관리, 예외 처리 학습

---

## 기능 구현 전 기능 목록
- [ ] 자동차 이름 입력 기능 구현 (쉼표 구분, 최대 5자)
- [ ] 자동차 이름 입력 예외 처리 (빈 문자열, 공백, 5자 초과, 잘못된 구분자)
- [ ] 시도 횟수 입력 기능 구현
- [ ] 시도 횟수 입력 예외 처리 (0 이하, 음수, 숫자 아님)
- [ ] 랜덤 값 기반 전진 로직 구현 (0~9 랜덤 ≥ 4)
- [ ] 자동차 이동 상태 출력 기능 구현
- [ ] 우승자 계산 기능 구현 (단독/공동)
- [ ] 최종 우승자 출력 기능 구현
- [ ] 테스트 작성 (게임 로직 및 예외 테스트)

---

## 기능 요구 사항
- 자동차 이름 입력 가능 (쉼표 구분, 최대 5자)
- 시도 횟수 입력 가능
- 전진 조건: 0~9 랜덤 값 ≥ 4
- 우승자 계산 (단독/공동)
- 잘못된 입력 시 `IllegalArgumentException` 발생

---

## 입력/출력 예시

### 입력
```

경주할 자동차 이름을 입력하세요.(쉼표로 구분)
pobi,woni,jun
시도할 횟수는 몇 회인가요?
5

```

### 출력
```

실행 결과
pobi : -
woni :
jun : -

pobi : --
woni : -
jun : --

...
최종 우승자 : pobi, jun

````

## 프로그래밍 요구 사항
- JDK 21에서 실행 가능
- 프로그램 시작점: `Application.main()`
- 외부 라이브러리 사용 금지
- Indent depth ≤ 2
- 메서드 한 가지 역할만 수행
- 삼항 연산자 사용 금지
- 테스트 작성 필수

---

## 테스트
- **JUnit 5 + AssertJ** 사용
- 테스트 항목
  1. 자동차 이름 입력 예외
     - 5자 초과, 빈 문자열, 공백, 잘못된 구분자
  2. 시도 횟수 입력 예외
     - 0, 음수, 숫자가 아닌 문자열
  3. 게임 로직
     - 전진 조건, 단독/공동 우승자, 여러 회 경주, 모든 자동차 정지
- 테스트 실행 방법:
```bash
./gradlew clean test  # Mac/Linux
.\gradlew.bat clean test  # Windows
````
